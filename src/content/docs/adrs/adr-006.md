**Status:** Accepted — 2026-07-09

## Context
a unified ecosystem for the database (PostgreSQL), file storage, and authentication out of the box.

## Decision
We selected Supabase (managed Postgres, Auth, Storage, and Realtime) as the backend operating environment for the CSRP MVP.
 
Supabase provides a unified platform for relational data, authentication, file storage, and real-time subscriptions, reducing the number of custom subsystems required for an MVP timeline, and its Realtime layer (built on Postgres logical replication) aligns naturally with the platform's live, reasoning-first quiz sessions.

Because the underlying data store is standard Postgres, migration away from the platform remains low-risk if scale or compliance requirements change. Row Level Security (RLS) is used to enforce per-role and per-classroom data access policies directly at the database layer, supporting the platform's privacy-by-design posture for minor users. 

## tradeoff 
Supabase does not currently offer an Africa-based region (af-south-1 is unsupported), so the project will be deployed to the nearest available region (EU — Frankfurt/London/Paris), introducing higher latency (~150–200ms) for Kenyan users and cross-border processing of personal data. This is considered acceptable for MVP under Kenya's Data Protection Act 2019, which permits cross-border transfer with adequate safeguards, but will be revisited if the platform moves beyond pilot scale. Parental-consent and age-verification logic for minors is not provided out of the box by Supabase Auth and will be implemented at the application layer.

## Revisit triger
If the scalling outgrows the supabase platform
